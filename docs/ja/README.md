# OER Exchange — 利用者向けドキュメント

`local_oerexchange` は、OER Exchange プラットフォームのカタログ兼ウェブサービス
API プラグインです。教員自身の Moodle サイトから共有されたオープン教育リソース
(OER)を公開・閲覧・プレビュー・レビュー・インポートするための中心的なサイトに
インストールします。1 つの専用「Exchange」サイトにインストールし、教員は直接
(閲覧・レビュー)、または自分の Moodle サイトにインストールした
[`local_oerclient`](https://github.com/adamjenkins/moodle-local_oerclient)
プラグイン経由でやり取りします。

本書は開発者向けではなく、サイトを**利用する**方向けのドキュメントです。
実装の詳細はリポジトリ自体のソースおよびコミット履歴を参照してください。

## 誰でも: カタログの閲覧と利用

`/local/oerexchange/index.php` のカタログは匿名の訪問者にも公開されています —
検索・閲覧にログインは不要です。

- キーワード、リソースの種類(コース全体か単一アクティビティか)、ライセンス、
  言語で**検索・絞り込み**ができます。
- カードをクリックすると**リソースページ**が開き、以下が表示されます。
  - **構造プレビュー** — 共有されたバックアップから生成された、コースの
    セクションとアクティビティの一覧。誰かが先にインポートしなくても内容が
    わかります。
  - 共有時に指定された**ライセンス**(クリエイティブ・コモンズまたは
    パブリックドメイン)。
  - **必要なプラグイン** — 標準の Moodle に含まれないアクティビティタイプや
    コース形式を使っている場合に表示されます。それぞれ「トライアルに含まれて
    います」(インストールなしで試せます)、または「トライアルには含まれて
    いません — 試したときには表示されません」(インポート前に自分のサイトに
    そのプラグインを入れておく必要があるか、そのアクティビティが省略される
    ことを許容する必要があります)のいずれかが表示されます。
  - 他の教員による**アダプテーション・ストーリー(実践レビュー)** — どう
    変更してどう活用したか。
  - **ダウンロード数**と**インポート数**。そのリソースが別のリソースを
    改変したものであれば、出典へのリンクも表示されます。
- **Try it(試してみる)**をクリックすると、ブラウザ内だけで完結する使い捨ての
  フル Moodle 環境でそのリソースが開きます(アカウント不要、サーバー側での
  実行なし、何も保存されません)。インポートする前に自分のコースに合うか
  確かめる最も速い方法です。サイト管理者がサンドボックスを有効にしていて
  (下記参照)、リソースの処理が完了している場合のみ表示されます。
- **Download(ダウンロード)**は、Moodle 標準のコース復元画面から手動で
  インポートしたい場合に、バックアップファイルそのものを直接取得します。

## 教員向け: レビューと報告

レビューや報告を行うには、ログインした上で個人の Exchange アカウントと連携
している必要があります(通常は、自分のサイトの `local_oerclient` プラグインで
初めて共有・操作しようとしたときに自動的に案内されます — 「Exchange
アカウントを連携する」の手順は同プラグインのドキュメントを参照してください)。

- **アダプテーション・ストーリーを共有する**: 任意のリソースページで、
  どのように使ったか、何を変更したか、結果はどうだったかを記入します。
  これはこのプラットフォームの核となる考え方です — 共有とは単にファイルを
  アップロードすることではなく、次に使う教員の改変を助けることでもあります。
  星による任意の評価も可能です。
- **報告(Report)**: 著作権上の懸念、品質の問題、スパム、その他の理由で
  リソースにフラグを立てます。報告はモデレーションキューに送られます —
  報告してもリソースがすぐに消えるわけではありませんが、確認は行われます。

## サイト管理者向け

### 初期設定

プラグインをインストール後(**サイト管理 ▸ 通知**からアップグレードの案内に
従ってください)、**サイト管理 ▸ プラグイン ▸
OER Exchange** を開いて以下を設定します。

- **サンドボックスを有効化(Try it)** — カタログ全体で「Try it」ボタンを
  表示します。Moodle Playground のデプロイ先を指定する必要があります(下記)。
- **サンドボックスのベース URL** — Moodle Playground の静的ビルドが
  デプロイされている、同一オリジンのパス。例:
  `https://your-exchange.example/try/`。このデプロイ自体はサーバー管理の
  作業であり本書の対象外です — プラットフォームの `oer-sandbox` デプロイ
  キットのドキュメントを参照してください。

`local/oerexchange:moderate`(報告と解析失敗のモデレーション権限)と
`local/oerexchange:managesites`(クライアントサイトの承認・プラグイン許可
リストの管理権限)の 2 つのケーパビリティで権限を制御します。どちらも
既定で **Manager(管理者)** ロールに付与されています。

### クライアントサイトの承認

ある Moodle サイトが自身の `local_oerclient` プラグイン(「Exchange に
登録する」ページ)からこの Exchange に登録すると、**サイト管理 ▸
プラグイン ▸ OER Exchange ▸ 登録済みサイト** の
「保留中」欄に表示されます。

1. サイト名、URL、連絡先メールアドレスを確認します。
2. **承認(Approve)** をクリックします。これにより、その サイト専用の
   非対話型アカウントが Exchange 上に作成され、そのアカウント用の
   ウェブサービストークンが発行されます — サイトの「キー」の実体はこの
   トークンであり、別途パスワードがあるわけではありません。
3. トークンは連絡先アドレスにメール送信されます(このサーバーで送信メールが
   設定されている場合)。また承認画面に一度だけ表示されます。**別の方法で
   伝える必要がある場合はすぐにコピーしてください** — 復元可能な形では
   保存されず、二度と表示されません。
4. クライアントサイトの管理者は、このトークンを自分の `local_oerclient`
   の設定(「サイトトークン」)に貼り付けることで連携が完了します。

**取り消し(Revoke)** はサイトのアクセス権を即座かつ恒久的に無効化します —
そのサイトが保持するすべてのトークンが直ちに使えなくなります。取り消した
サイトを再承認すると、同じアカウントを再利用して新しいトークンを発行します。

### サンドボックス・プラグイン許可リストの管理

共有されたリソースの中には、標準の Moodle に含まれないコントリビュート
アクティビティモジュールやコース形式を使っているものがあります。サンドボックス
(「Try it」)には、サイト管理者が明示的に選定してアップロードしたプラグイン
のみを含めることができます — これは意図的な安全境界です。リソースの
メタデータだけで訪問者のブラウザに任意のコードをインストール・実行させる
ことは決してできません。

**サイト管理 ▸ プラグイン ▸ OER Exchange ▸
サンドボックス・プラグイン許可リスト** で:

1. プラグインタイプ(例:`mod`)、プラグイン名(タイププレフィックスを
   除いたフランケンスタイル名、例:`board`)、対象の Moodle ブランチ
   (例:`5.2`)を入力します — これはサンドボックスのデプロイが実際に
   提供しているブランチのいずれかと一致している必要があります。
2. プラグインのリリース ZIP をアップロードします(プラグイン自身の GitHub
   リリース、または camp-registry.org のようなコミュニティレジストリから
   入手できます)。
3. 追加すると、必要プラグイン一覧がトライアルのブランチに対する有効な
   許可リストの項目と一致するリソースは、誰かが Try it をクリックした際に
   そのプラグインが自動的にインストールされます。項目は削除せずに
   無効化することもできます。

### モデレーション

**サイト管理 ▸ プラグイン ▸ OER Exchange ▸
モデレーションキュー**(`local/oerexchange:moderate` が必要)には以下が
表示されます。

- **未処理の報告** — 「解決(Resolve)」(対応済みとしてマーク)または
  「却下(Dismiss)」(対応不要)を選択できます。**「非表示(Hide)」**は
  リソースを削除せずに即座に公開カタログから外します。
- **解析に失敗したもの** — アップロードされたバックアップを処理できな
  かったリソース(構造プレビューも「Try it」も利用不可)。エラーメッセージ
  が表示されるので、共有した教員に連絡すべきか判断する際の参考になります。

## プライバシーとデータの取り扱い

Exchange は次の個人データを保持します: あなたが書いたレビュー、あなたが
行った報告、あなたが共有したリソース(あなたのアカウントに紐づけ)、
インポートおよびサンドボックス・トライアルの記録、そしてアカウント連携の
処理中に短時間だけ存在するワンタイムコード。これらはすべて本プラグインの
GDPR プライバシープロバイダーの対象です — データのエクスポートや削除には
Moodle 標準の**サイト管理 ▸ ユーザー ▸ プライバシーとポリシー ▸
データリクエスト**機能を利用してください。共有されたリソース自体は
リクエストによって削除されません(他のサイトが既にインポートしている
可能性があるため)— 代わりに、あなた個人への帰属情報が削除されます。

共有用にアップロードされるすべてのバックアップは、公開前にサーバー側で
独立して「ユーザーデータを含まないこと」が検証されます。アップロード元の
サイトが何を主張していたかにかかわらず検証されます。

## トラブルシューティング

- **リソースに「Try it」が表示されない** — サイト全体でサンドボックスが
  有効になっていないか(管理者設定)、そのリソースのバックアップの解析が
  まだ完了していません(解析失敗がないかモデレーションキューを確認して
  ください)。
- **必要なプラグインに「トライアルには含まれていません」と表示される** —
  サイト管理者がそのリソースの Moodle バージョン用にサンドボックス・
  プラグイン許可リストへ追加していません。追加を希望する場合は管理者に
  連絡するか、インポート前に自分のサイトにそのプラグインをインストール
  してください。
- **ダウンロードリンクがない** — リソースの処理がまだ完了していないか、
  モデレーションにより非表示・削除されています。

## 謝辞

「Try it」は **[Moodle Playground](https://github.com/ateeducacion/moodle-playground)**
によって実現されています — ブラウザ上で完全に動作する使い捨ての Moodle
インスタンスは、このプラグインの成果ではなく彼らの成果です(WASM 起動、
ブループリントによるプロビジョニング、Service Worker など)。このプラグイン
が行っているのは起動用 URL の構築のみです。Moodle Playground 自体は
**[WordPress Playground](https://github.com/WordPress/wordpress-playground)**
の PHP-in-WebAssembly ランタイム上で動作しています。
